해시 테이블

해시함수를 사용하여 변환한 값을 index로 삼아 key와 데이터를 저장하는 자료구조이다.
기본 연산으로는 탐색, 삽입 삭제가 있다.

적재율(Load Factor)
해시 테이블의 크기 대비 키의 개수를 말한다.
즉, 키의 개수 = K, 테이블의 크기 = N 일때 K/N이다.


충돌(Collision)
서로 다른 내용의 데이터가 같은 키를 가지는 경우를 말한다.
즉 이는 특정 버켓에 데이터가 집중된다는 뜻이다.
따라서 충돌이 많이 일어나면 해시 테이블의 성능이 떨어진다.
크게 중돌을 다루는 방법은 다음과 같다.

1. 체이닝(Chaining)
충돌이 발생할 경우 동일한 버킷에 연결리스트를 이용하여 저장한다.
삽입 - O(1)
삭제, 탐색 - O(K) -> 최악의 경우 적재율을 이용하여 표현

2. 개방 주소법(Open Addressing)
충돌이 발생할 경우 다른 주소로 이동하여 저장한다.

2-1 선형 탐사(Linear Probing)
충돌이 발생한 경우 인접한 인덱스에 데이터를 삽입한다.
클러스터링 문제가 발생한다.

2-2 제곱탐사(Quadratic Probing)
충돌이 발생한 경우 제곱의 인덱스에 데이터를 삽입한다.
이 역시 초기 해시값이 같은 경우 클러스터링 문제가 발생한다.
